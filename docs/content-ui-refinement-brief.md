# 홈·프로젝트·이력 콘텐츠 개편 명세

## 범위와 근거

현재 `index.html`, `projects.html`, `i18n.js`, `_config.yml`에 확인되는 내용만 사용한다. 확인되지 않은 학교, 전공, 학위, 재학 기간, 연구실명, 프로젝트 URL은 만들지 않는다.

## 1. 관심사

홈의 관심사 영역은 연구 질문 목록보다 지금 관심을 두는 주제를 소개하는 영역으로 쓴다.

- 섹션 레이블: `Interests` (한영 공통)
- 제목: `요즘 관심 있는 것` / `What I'm interested in`
- 항목: 기존 세 주제인 mechanistic interpretability, LLM 및 NLP, AI agent와 도구 개발을 유지한다.

항목은 새 활동이나 성과를 덧붙이지 않고 기존 설명을 그대로 사용한다. 제목과 레이블에서만 연구에 한정된 인상을 덜어낸다.

## 2. Patent와 Software

Patent와 Software는 내비게이션, 홈, 프로젝트, 이력 어디에도 별도 섹션이나 항목으로 노출하지 않는다. 현재 페이지와 한영 사전에는 두 항목이 없으므로 새로 만들거나 빈 섹션을 추가하지 않는다.

## 3. 학교와 연구실

기존 `학력` / `Education` 대신 `학교와 연구실` / `Education & labs`를 사용한다. 한 항목은 다음 순서로 표시한다.

1. 재학 기간 / Dates attended
2. 학교와 전공 또는 학위 / School and major or degree
3. 소속 연구실 / Lab

현재 저장소에서 확인되는 값은 `KAIST SAIL`뿐이다. 이를 학교명이나 학위 정보로 풀어 쓰지 않는다. 나머지는 관리 화면이나 초안에서 아래처럼 명시한다.

- `재학 기간: TODO` / `Dates attended: TODO`
- `학교 · 전공/학위: TODO` / `School · major/degree: TODO`
- `연구실: KAIST SAIL` / `Lab: KAIST SAIL`

공개 화면을 구현할 때는 TODO가 남은 필드를 숨기지 말고 미확정 정보임을 그대로 보여 준다. 실제 정보가 확인되면 ko/en 값을 함께 바꾼다.

## 4. 프로젝트 외부 링크

프로젝트마다 선택형 `links` 목록을 둘 수 있다. 허용하는 종류와 표시 순서는 다음과 같다.

1. `demo`: `데모` / `Demo`
2. `repository`: `저장소` / `Repository`
3. `paper`: `논문` / `Paper`
4. `details`: `자세히` / `Details`

각 링크는 `type`, 실제 `url`, 선택형 ko/en 접근성 설명을 가진다. URL이 확인된 링크만 설명 아래에 위 순서로 표시한다. 값이 없으면 링크와 빈 링크 영역을 모두 렌더링하지 않는다. 홈의 프로젝트 미리보기에는 외부 링크를 넣지 않고 `/projects/`의 전체 카드에서만 표시한다.

현재 저장소에는 각 프로젝트에 연결할 실제 외부 URL이 없다. 따라서 이번 변경에서는 링크를 노출하지 않는다. 프로필용 GitHub 주소를 프로젝트 저장소 주소로 재사용하지 않는다.

## 5. 테마와 언어 버튼 문구

컨트롤은 아이콘만 쓰지 않고 짧은 텍스트가 보이는 버튼으로 만든다.

| 컨트롤 | 한국어 화면 | 영어 화면 | 상태·동작 안내 |
|---|---|---|---|
| 테마 | `테마` | `Theme` | 밝은 상태: `어두운 테마로 전환` / `Switch to dark theme`; 어두운 상태: `밝은 테마로 전환` / `Switch to light theme` |
| 언어 | `EN` | `한국어` | 한국어 상태: `Switch to English`; 영어 상태: `한국어로 전환` |

테마 버튼의 `aria-pressed`는 어두운 테마일 때 `true`로 둔다. 버튼을 누른 뒤에는 테마 상태와 `aria-label`을 함께 갱신한다. 언어 버튼의 보이는 레이블은 현재 언어가 아니라 전환될 언어를 가리킨다.

## 확인 항목

- 관심사 제목과 레이블이 ko/en에서 새 문구로 바뀐다.
- 기존 세 관심사 항목의 의미는 달라지지 않는다.
- Patent와 Software 문구가 공개 페이지와 i18n 키에 없다.
- 학교 또는 학위를 KAIST로 추정한 문구가 없다.
- 프로젝트 URL이 없는 상태에서 빈 링크나 임시 링크가 보이지 않는다.
- 테마와 언어 버튼은 보이는 짧은 레이블과 상태에 맞는 접근성 문구를 가진다.
- ko/en i18n 키 집합이 일치한다.
